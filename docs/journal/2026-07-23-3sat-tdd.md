## Command note and CI intent

The canonical project and CI command is:

```powershell
pytest -q
```

On this developer machine, the `pytest` console-script directory was not on
`PATH` (the direct `pytest -q` invocation was not recognized). The equivalent
interpreter invocation, `python -m pytest -q`, was used locally so that the
same installed pytest package and project configuration executed. CI should use
the canonical `pytest -q` command after installing the development dependency.
